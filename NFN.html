Non-Functional Requirements

Security

  1. The database ought to be secured against intrusions and illegal access.

  2. For security, HTTPS will be used to establish the connection between the Web API and the apps.

Effectiveness

  1. If exceptions do occur, they should be effectively reported to the user.

Scalability

  1. The system must be scalable depending on how many people are utilising it.

Availability

  1. Our programme is fully independent of any network provider and can operate even when there is no internet connection. It can be used every day of the year, 24/7.

  2. The chatbot ought to be designed to function in the majority of server configurations. In light of this, a chatbot should be able to function in either server environment.

  3. The chatbot must be able to understand natural language within a certain domain and be able to determine the user input topic independently of the construction of the sentence.

Performance

  1. The chatbot should write answers more quickly than it would take a human to do so.

  2. Over the question testing set, the server will typically respond in less than or equal to 2 seconds.

Data management

  1. In order to efficiently address customer issues and obtain insights from pertinent data, the chatbot can acquire, interpret, and process massive amounts of data.

User Interface

  1. User interfaces and experiences (UI / UX) must be carefully created.

Expandability

  1. By examining interactions, chatbot software should constantly increase its own knowledge base.
